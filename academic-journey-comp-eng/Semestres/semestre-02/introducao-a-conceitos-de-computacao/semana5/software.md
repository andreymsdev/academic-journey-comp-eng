## Software

#### Algoritmo, programa e software

* **Algoritmo:** É um passo a passo lógico, finito e ambíguo para resolver um problema.
* **Programa:** É a implementação de um algoritmo em uma linguagem que o computador consiga processar.
* **Software:** Conjunto completo que engloba um ou mais programas, documentação e estruturas de dados necessários para realizar tarefas complexas e resolver problemas do usuário.

#### Tipos de Software

* **Software de Sistema (Básico):** Fornece a plataforma para outros softwares (ex: Sistemas Operacionais como Windows/Linux, drivers e compiladores).
* **Software de Aplicação:** Programas para o usuário final (ex: navegadores, editores de texto, jogos).
* **Software de Programação:** Ferramentas que ajudam desenvolvedores a criar outros programas (ex: IDEs, depuradores).
* **Software Embarcado:** Softwares que residem dentro de dispositivos de hardware específicos (ex: sistemas de micro-ondas, controladores automotivos).
* **Plugins/Extensões:** Softwares secundários que adicionam funcionalidades específicas a um programa principal.

#### Linguagem de programação

* **Imperativo:** Foca em *como* a tarefa deve ser feita, descrevendo estados e comandos para mudar esses estados (ex: C, Pascal).
* **Funcional:** Trata a computação como a avaliação de funções matemáticas e evita estados mutáveis (ex: Haskell, Lisp).
* **Lógico:** Baseado em lógica formal; o programador define fatos e regras, e o sistema deduz a solução (ex: Prolog).
* **Orientado a Objetos (POO):** Organiza o código em "objetos" que contêm dados (atributos) e procedimentos (métodos) (ex: Java, Python, C++).

---

O ciclo de vida (SDLC - Software Development Life Cycle) é a estrutura que define as etapas do desenvolvimento, desde a concepção até o descarte. As fases principais são:

1.  **Requisitos:** Entender o que o software precisa fazer.
2.  **Análise e Design:** Planejar a arquitetura e a interface.
3.  **Codificação (Implementação):** Escrever o código propriamente dito.
4.  **Testes:** Verificar se há erros e se o sistema atende aos requisitos.
5.  **Implantação:** Lançar o software para o usuário final.
6.  **Manutenção:** Corrigir bugs e adicionar melhorias ao longo do tempo.
