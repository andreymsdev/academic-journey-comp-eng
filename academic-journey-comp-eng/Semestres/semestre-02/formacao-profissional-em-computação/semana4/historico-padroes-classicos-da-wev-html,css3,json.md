## História

**Vannevar Bush** é considerado o inventor do conceito de "hipertexto" em 1945. 
O memex deveria imitar a forma de recuperação de informações na mente humana.
- Documentos convencionais correspondem a versão linear dessas informações.

**Ted Nelson** cunhou o termo hipertexto.
- Propôs o sistema hipertexto Xanadu que deveria suportar o acesso e o armazenamento de hiperdocumentos em todo o mundo de modo transparente, forma similar de como trabalha a WWW hoje.

**Douglas Engelbart** é considerado o pai da computação interativa e o inventor do mouse.
- Pioneiro ao demostrar uma estação de trabalho com ambiente de janelas, na qual o sistema distríbuido permitia a comunicação entre os usuários remotos e a interação desses usuários com um hiperdocumento armazenado remotamente.

## Rastro do começo da WEB

**Tim Berners-Lee** apresenta junto ao CERN, a proposta de seu sistema **MESH**.
- O MESH pretende suportar:
    - Uma arquitetura cliente-servidor.
    - Endereçamento universal (qualquer usuário pode ter acesso à informação)
    - Diversos tipos de documentos atráves de diversos protocolos.

Em 1990 apresenta o protopido de um servidor que passa a ser chamada de **World Wide Web**. Em 1991 o servidor é disponibilizado para computadores do CERN. Em 1992, interfaces cliente e servidor são disponibilizadas publicamente e Berners-Lee prooduz a primeira formalização do **HTML**.

- **1994** surge a World Wide Web Consortium (W3C) formado por universidades, com o obketivo de promover um direcionamento estável para a WEB.

| Padrões Clássicos da Web  |                |
|---------------------------|----------------|
| HTML                      | CSS            |
| JavaScript                | XML            |
| Ajax                      | JSON           |

**AJAX** usa JS para enviar e receber dados, dados trocados em XML. 
**JSON** formatação para troca de dados.

**Observações:**

- Ajax → Fetch API: o ```fetch()``` substitui o antigo ```XMLHttpRequest```, mas a ideia de AJAX continua viva.
- JSON vs XML: JSON se tornou o padrão dominante porque é mais leve e integrado ao JavaScript.
